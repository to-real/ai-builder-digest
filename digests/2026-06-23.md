## Training Data — Google DeepMind's Logan Kilpatrick: Why the Model Eats the Harness

*Training Data*

**The "model" is no longer just weights but an expanding system that eventually absorbs external scaffolding like agent harnesses.**

Logan Kilpatrick, who runs Google AI Studio and the Gemini API, argues that the definition of a "model" has evolved from a static set of weights into a sprawling system including tools, search, and code execution. He observes that external scaffolding—like agent harnesses—often leads the way in capability, but the "model eats the harness" over time as these features are upstreamed into the native system. He predicts that the current focus on building custom harnesses may be temporary as models become more generalized. On coding, he views current coding agents as a form of "narrow superintelligence" that accelerates human ambition rather than replacing it, and notes that Google is aggressively dogfooding Gemini internally to drive this progress. "The scaffolding is oftentimes a couple of steps ahead of what is baked directly into the model, and then what ends up happening is the model eats that scaffolding and it becomes part of the native model system."

Google AI Studio 和 Gemini API 的负责人 Logan Kilpatrick 认为，“模型”的定义已经从一组静态的权重演变为一个包含工具、搜索和代码执行的庞大系统。他观察到，外部脚手架（如 agent harness）通常在能力上领先，但随着这些功能被原生系统吸收，“模型会吞噬框架”。他预测，随着模型变得更加通用，目前专注于构建自定义 harness 的趋势可能是暂时的。在编码方面，他将当前的 coding agent 视为一种“狭义超级智能”，它加速了人类的雄心而不是取代人类，并指出 Google 正在内部积极地使用 Gemini 来推动这一进展。“脚手架往往领先于直接内置在模型中的功能几步，最终发生的情况是模型吞噬了这些脚手架，使其成为原生模型系统的一部分。”
